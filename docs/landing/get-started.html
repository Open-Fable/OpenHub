Get started

Up and running in two minutes.

Download the app, add your API keys once, and every tool is ready to go. No account, no sign-up. No background services left running after you quit.

Check your system

OpenHub requires macOS 14 (Sonoma) or later and an Apple Silicon Mac (M1, M2, M3, M4 or later). Intel Macs are not supported.

Download the app

Grab the latest .dmg from the GitHub releases page. The file is about 80 MB. No installer, no dependency to install.

Install it

Open the .dmg, drag OpenHub into your Applications folder. Then eject the disk image and launch OpenHub from Applications (or Spotlight).

Add your API keys

OpenHub ships with no AI credits. Open the app and click the Config tab in the sidebar. You'll see fields for each supported provider:

  • Anthropic — get a key at console.anthropic.com
  • OpenAI — get a key at platform.openai.com
  • OpenRouter — get a key at openrouter.ai
  • Ollama — runs 100 % local, no key needed

Keys are stored in an encrypted file at ~/Library/Application Support/openhub/secrets.enc (AES-256-GCM), not in plaintext on disk. A local proxy on 127.0.0.1:9999 holds them in memory and routes every request. The tools only ever see a fake local token.

Start creating

That's it. Click any tool in the sidebar: Chat to talk to a model, Code to open the code agent, Design for mockups, Work for documents, or Orchestrator to hand a goal to a team of agents. One config works everywhere.

Troubleshooting

Something not working?

The app won't open / macOS says it's damaged

OpenHub is signed and notarized. If Gatekeeper blocks it, go to System Settings → Privacy & Security, scroll to the "OpenHub was blocked" message, and click Allow Anyway. Then open it again.

Proxy isn't starting / "Connection refused"

The proxy binds to 127.0.0.1:9999. Make sure another service isn't already on that port. Check the app logs: ~/.config/openhub/logs/. If the port is free and it still fails, open a GitHub issue.

One of the wrapped tools won't load

OpenHub starts OpenCode, OpenWork, and Open Design as separate processes. If a tool tab stays blank, quit OpenHub and relaunch. If the problem persists, run npm run update:apps in the project directory to rebuild the tool bundles.
