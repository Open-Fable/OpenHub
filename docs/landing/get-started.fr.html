Démarrer

Prêt en deux minutes chrono.

Téléchargez l'appli, ajoutez vos clés API une seule fois, et tous les outils sont prêts. Pas de compte, pas d'inscription. Aucun service qui tourne après avoir quitté.

Vérifiez votre système

OpenHub nécessite macOS 14 (Sonoma) ou ultérieur et un Mac Apple Silicon (M1, M2, M3, M4 ou ultérieur). Les Mac Intel ne sont pas supportés.

Téléchargez l'application

Récupérez le dernier .dmg sur la page des releases GitHub. Le fichier fait environ 80 Mo. Pas d'installateur, pas de dépendance.

Installez

Ouvrez le .dmg, glissez OpenHub dans votre dossier Applications. Éjectez l'image disque puis lancez OpenHub depuis Applications (ou Spotlight).

Ajoutez vos clés API

OpenHub ne contient pas de crédits IA. Ouvrez l'app et cliquez sur l'onglet Config dans la sidebar. Vous y trouverez les champs pour chaque fournisseur :

  • Anthropic — clé sur console.anthropic.com
  • OpenAI — clé sur platform.openai.com
  • OpenRouter — clé sur openrouter.ai
  • Ollama — 100 % local, pas de clé nécessaire

Les clés sont stockées dans un fichier chiffré au chemin ~/Library/Application Support/openhub/secrets.enc (AES-256-GCM), pas en clair sur le disque. Un proxy local sur 127.0.0.1:9999 les garde en mémoire et achemine chaque requête. Les outils ne voient qu'un faux jeton local.

Commencez à créer

C'est tout. Cliquez sur un outil dans la sidebar : Chat pour parler à un modèle, Code pour l'agent de code, Design pour des maquettes, Work pour des documents, ou Orchestrateur pour confier un objectif à une équipe d'agents. Une seule config pour tous.

Problèmes fréquents

Quelque chose ne marche pas ?

L'app ne s'ouvre pas / macOS dit qu'elle est endommagée

OpenHub est signé et notarisé. Si Gatekeeper la bloque, allez dans Réglages Système → Confidentialité & Sécurité, descendez jusqu'au message « OpenHub a été bloqué », et cliquez sur Autoriser quand même. Rouvrez ensuite l'application.

Le proxy ne démarre pas / « Connexion refusée »

Le proxy écoute sur 127.0.0.1:9999. Vérifiez qu'aucun autre service n'occupe ce port. Consultez les logs : ~/.config/openhub/logs/. Si le port est libre et que ça ne marche toujours pas, ouvrez un signalement GitHub.

Un des outils intégrés ne se charge pas

OpenHub lance OpenCode, OpenWork et Open Design dans des processus séparés. Si un onglet reste vide, quittez et relancez OpenHub. Si le problème persiste, lancez npm run update:apps dans le répertoire du projet pour reconstruire les bundles des outils.
