FAQ

Les questions que tout le monde se pose d'abord.

Couts, plateformes, ou vont tes donnees, ce qu'est vraiment l'orchestrateur. Des reponses directes, sans detour.

OpenHub est-il gratuit ?

L'enveloppe OpenHub est open source sous licence MIT, donc oui, l'app elle-meme est gratuite. Elle n'inclut aucun credit IA, par contre : tu apportes tes propres cles API (Anthropic, OpenAI, OpenRouter, Google) ou tu fais tourner des modeles locaux via Ollama. L'inference est facturee par ton fournisseur, pas par OpenHub.

Ca tourne sur Windows ou Linux ?

Pas encore, macOS 14 et plus uniquement. OpenHub s'appuie sur le Trousseau macOS pour stocker les secrets et sur le systeme de vues d'Electron pour la sidebar. Un portage vers d'autres plateformes n'est pas prevu a court terme.

Ou sont stockees mes cles API ?

Dans le Trousseau macOS. Jamais ecrites sur le disque, jamais en local storage, jamais dans les vues des outils. Le proxy les charge en memoire et les injecte au demarrage des process. Les trois outils embarques ne recoivent qu'un faux token local, jamais tes vraies cles.

Ou sont mes donnees, projets, memoire, chats, stockees ?

Tout est local, dans ton dossier perso : projets et memoire sous ~/.config/openhub/, la config des outils sous ~/.config/opencode/, et l'historique de chat en local storage avec une sauvegarde fichier sur le disque. Rien n'est envoye a un serveur OpenHub, parce qu'il n'y en a pas. Le detail complet est sur la page confidentialite.

Quels fournisseurs LLM sont supportes ?

Anthropic (Claude), OpenAI (GPT), OpenRouter (multi-modeles), Ollama (local) et Google Gemini. Tu peux melanger cloud et local selon la tache, le tout par le meme proxy.

L'orchestrateur, c'est quoi concretement ?

Un moteur qui decoupe un objectif en agents specialises, code, design, work, recherche, verification, organises en graphe de dependances, puis les execute avec des controles qualite automatiques et deterministes. Le resultat est un dossier de vrais livrables, pas un journal de chat. L'histoire complete est sur la page outils et dans la reference du moteur.

Les apps open-source embarquees sont-elles modifiees ?

Jamais. OpenHub clone OpenWork, OpenCode et Open Design et les laisse intactes. Toute la personnalisation, le theme, les reglages natifs masques, les fonctions ajoutees, vit dans une couche d'injection separee. C'est pour ca que les mises a jour upstream ne cassent rien.

Pourquoi un proxy local sur le port 9999 ?

Pour centraliser tes secrets et ta config au meme endroit. Au lieu de coller tes cles dans trois apps, tu les entres une fois ; le proxy (127.0.0.1:9999, compatible OpenAI, Bearer requis) detient les vraies cles et route chaque appel vers le bon fournisseur. C'est aussi la que la recherche web et l'injection de contexte ont lieu.

L'interface a casse apres une mise a jour. Je fais quoi ?

Lance la verification des selecteurs (npm run check:selectors). Elle verifie que les selecteurs CSS sur lesquels les overrides s'appuient existent encore dans le nouveau code des apps, et signale ceux qu'un changement upstream a deplaces, pour que tu saches exactement quel override ajuster.

Comment contribuer, ou signaler un bug ?

Les bugs et demandes de fonctionnalites passent par les issues GitHub, avec un template pour chaque. Pour contribuer du code, vois CONTRIBUTING.fr.md : fork, branche, lance les tests et les verifications, ouvre une pull request.

Toujours bloque ? Ouvre une issue sur GitHub ou lis le guide d'usage complet. Chaque commentaire recoit une reponse.

Telecharge. Ajoute tes cles. Cree.

Glisse OpenHub dans ton dossier Applications, puis ajoute tes cles API dans le panneau Config. Apple Silicon, macOS 14 et plus.
