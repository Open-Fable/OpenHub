# OpenHub Launch Checklist

## Before launch

- [ ] Record a 30-second GIF/video of the app in action (switching slots, chat, orchestrator)
- [ ] Create a social preview image (1280x640) and upload to GitHub repo settings
- [ ] Set GitHub topics and description (see `github-setup.md` commands)
- [ ] Enable GitHub Discussions
- [ ] Enable GitHub Sponsors
- [ ] Pin repo on your GitHub profile
- [ ] Set up GitHub Pages for the landing page (see `landing-page/` or enable from Settings → Pages → `docs/` folder)
- [ ] Verify the .dmg download works from the Releases page
- [ ] Double-check all links in README (upstream repos, docs, etc.)

## Launch day (Tuesday–Thursday)

### Morning (before 14:00 UTC)

- [ ] Post on Hacker News — Show HN (see `show-hn.md`)
- [ ] Monitor HN comments, reply to all within first 2 hours

### Afternoon (14:00–17:00 UTC)

- [ ] Post Twitter/X thread (see `twitter-thread.md`)
- [ ] Post Bluesky thread (see `bluesky-thread.md`)
- [ ] Tag upstream project maintainers in the threads

### Evening (staggered, 4–6h apart)

- [ ] Post on r/opensource (see `reddit-posts.md`)
- [ ] Post on r/LocalLLaMA
- [ ] Post on r/macapps
- [ ] Post on r/selfhosted

## Week 1 after launch

- [ ] Reply to every GitHub issue and Discussion
- [ ] Reply to every Reddit/HN comment
- [ ] Send a polite message to upstream maintainers (OpenCode, OpenWork, Open Design) about the integration
- [ ] Write a short blog post / dev.to article about the architecture

## Week 2+

- [ ] Submit PRs to awesome lists (see `awesome-lists.md`) — wait for 50+ stars
- [ ] Post on Product Hunt (see `product-hunt.md`)
- [ ] Consider a YouTube demo (5 min walkthrough)
- [ ] Monitor GitHub trending weekly
